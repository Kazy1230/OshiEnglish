# ManaVillage 要件定義書 v1.1

> ステータス: ドラフト v1.1
> 対象フェーズ: MVP(英語学習 AI伴走コース)
> バージョン: v1.1

---

## 1. サービス概要

### サービス名
ManaVillage

### コンセプト
**成果達成型メンタープラットフォーム**

クリエイターの学習哲学・指導方針・判断基準をAIが再現し、学習者に伴走体験を提供するプラットフォーム。

### ミッション
「誰もがメンターを持てる世界をつくる」

### MVPの検証仮説
「ユーザーは動画教材ではなく、伴走体験にお金を払うのか？」

---

## 2. MVPのスコープ

### 対象ジャンル
英語学習(TOEIC / 英検 / 留学)

### 対象クリエイター
英語系マイクロクリエイター(フォロワー300〜5,000人)

### 対象学習者
TOEIC学習者 / 英検受験者 / 留学希望者

### MVPで実装しないもの
- Tier1 コンテンツ単体販売
- Tier3 本人メンタリング
- コミュニティ機能
- スマホアプリ / プッシュ通知
- LINE・Discord連携
- 複数ジャンル展開

> 補足: 学習者単位の買い切り（一括）コース購入機能（Purchase）と、Tier A/B月額サブスクリプション機能（CourseSubscription）は両方実装済みで併存している。コースは `tier_a_price` / `tier_b_price` が設定されていればサブスク型、両方NULLの場合は買い切り型として扱われる。

## 3. ロール定義

| ロール | 概要 |
|---|---|
| 学習者 | コースを購入し、AI伴走を受ける |
| クリエイター | コースを作成・販売する |
| 管理者 | プラットフォームを管理する |

---

## 4. コースティア定義

MVPでは2種類のコースティアを提供する。

| ティア | 名称 | 回答者 | 価格帯 |
|---|---|---|---|
| Tier A | AI伴走コース | AIのみが回答 | 980〜1,980円/月 |
| Tier B | プレミアム伴走コース | AIに加え講師が1日1回直接回答 | 2,980〜5,000円/月 |

クリエイターはどちらのティアを提供するか選択できる。両方提供することも可能。

### 5.2 クリエイター申請フロー

初期フェーズでは品質管理を優先し、運営審査制を採用する。

| Step | 内容 |
|---|---|
| Step1 | クリエイター登録申請(未登録者は申請と同時にアカウント作成、既存学習者はログイン後に申請) |
| Step2 | プロフィール入力(専門分野・指導実績) |
| Step3 | SNSアカウント提出(YouTube / Instagram / X) |
| Step4 | 運営審査(実在性・専門性・ガイドライン適合性を確認) |
| Step5 | 承認 → コース作成権限付与 |

**ステータス遷移**: `CreatorProfile.status` は `pending`(審査待ち) → `active`(承認済み・コース作成可) または `suspended`(却下/利用停止)の3値で管理される。却下も停止も同じ`suspended`状態として扱われ、管理者はいつでも`suspended`→`active`に再開できる。

> 補足: すでに学習者として購入・サブスク契約履歴があるアカウントはクリエイター申請ができない(学習者とクリエイターの兼任を防止)。また、コース生成等のAI呼び出しを伴う機能(AIインタビュー・コンテンツ生成スタジオ等)は`status='active'`になるまで利用できない。

---

### 5.3 人格データ収集(AIインタビュー)

フォーム入力ではなく、AIが対話形式でクリエイターにインタビューし人格データを抽出する。

**フロー**
```
Step0: クリエイターが指導タイプ(共感型/指導型/激励型/厳格型)と性別(男性/女性/中性的)のプリセットを選択(任意)
Step1: AIがクリエイターに指導シナリオ質問(固定5問)
      例:「TOEIC学習者が単語学習を嫌がっています。どう指導しますか?」
Step2: クリエイターが自由回答
Step3: AIが必要に応じて深掘り質問(各固定質問につき最大3問まで追加)
Step4: AIが人格プロファイルとサンプル回答文(sample_reply)を生成
Step5: クリエイターが確認・修正して確定
```

所要時間目安: 10〜15分

途中保存・再開に対応しており、インタビューを中断しても`InterviewSession`に進行状況(現在の固定質問インデックス・深掘り回数・QA履歴)が保存され、続きから再開できる。

> **重要**: 1人のクリエイターには人格(キャラクター)が1つだけ存在する。複数の人格を持つことはできない。このインタビューが完了した時点（Step4〜5）で、人格プロファイルと同時に人格(キャラクター)データが自動的に作成される。クリエイターが別途「キャラクターを新規作成する」という操作は存在しない。

**抽出項目**

| カテゴリ | 項目 |
|---|---|
| コミュニケーション | 口調・一人称・語尾 |
| 指導スタイル | 厳しさ・励まし方・フィードバック方法 |
| 学習哲学 | 重視する考え方・判断基準・優先順位 |
| 思考特性 | 例え話の癖・説明方法・問題解決アプローチ |

### 5.4 コース作成機能

**クリエイターの入力**

| カテゴリ | 項目 |
|---|---|
| コース情報 | コース名・ゴール(goal)・対象学習者(target_learner)・コースティア(A/B) |
| 指導方針 | 学習強度(intensity)・使用教材(study_materials)・進行ペース(pace) |
| 教材 | 教材プリセット(管理者登録済み)または独自教材を追加し、目次(toc)単位で1〜30日への割り当てを設定 |
| Day1診断質問 | コースごとに独自のカスタム質問(`CourseDiagnosisQuestion`)を作成。固定の共通質問は廃止されている |
| 参考資料 | PDF・URL(コース生成には使用せず参考資料として添付) |

**AIが自動生成する内容**
- 30日学習計画(週単位テーマ + 日単位タスク。週ごとに複数回のAI呼び出しを要するためバックグラウンドタスクで生成し、`days_generation_status`(idle/generating/completed/failed)で進行状況を管理する)
- AI伴走設定(人格プロファイルを適用)

> コース作成画面に「キャラクターを選択する」という項目は存在しない。1クリエイター=1人格のため、コースは常にクリエイター本人の人格(キャラクター)に自動的に紐づく。

クリエイターはAI生成結果を日単位で確認・編集してから、セルフ品質チェック(5項目×20点、70点以上で「公開可」と判定)を経て公開申請(`draft`→`review`)する。運営が承認すると`published`になり公開される(却下されると`draft`に戻り、理由がメールで通知される)。

**教材プラン相談(AI教材プラン相談機能)**: クリエイターが教材の使い方を自然言語で説明すると、AIが各教材の1日あたりの学習量・周回目標・1〜30日への章割り当てを提案する。クリエイターは提案を確認のうえ確定(保存)する。

### 5.5 AI伴走コース(コア機能)

ManaVillageの最重要機能。

#### Day1: 初回診断(チャット形式)

固定の共通質問(現在スコア・目標スコア・試験日等)は廃止されており、Day1診断はクリエイターがコースごとに設定したカスタム質問(必須/任意を指定可能)と、コースに登録された教材ごとの進捗確認(未着手/進行中(周回数・進捗%)/完了)のみで構成される。

AIが実施:
- カスタム質問への回答と教材進捗をもとにレベル分析
- 30日ロードマップ生成(診断完了後すぐに提示)
- 学習者ごとの30日タスク個人化(教材の残り学習量を加味して各日のタスク量を調整。生成に失敗した場合は標準プランにフォールバックする)
- 通知時刻の設定(学習者が朝・夜の時刻を設定。デフォルトは07:00/21:00、「後で設定する」場合もデフォルト値で作成される)

Day1診断は1コースにつき1回のみで、再診断はできない。

#### Day2〜30: 日次伴走

| 機能 | 内容 |
|---|---|
| 学習報告 | 当日の学習内容を報告 |
| 相談・質問対応 | 学習上の疑問・悩みをAIに相談 |
| モチベーション支援 | AIが励まし・継続支援 |

#### 週次レビュー

Day番号が7の倍数(Day7, 14, 21, 28)になった時点でAIが自動生成する(1学習者・1コース・1週につき1回のみ)。
AIが分析: 当該週の完了/未完了日数・直近の質問カテゴリと苦手分野
出力: フィードバック + 翌週の学習計画。生成完了時にメール通知する。

#### 月次レビュー

Day30(MVPでは30日コースのため実質コース終了時点)にAIが自動生成する。
- ロードマップとの差分確認(目標スコア・レベル分析との比較)
- 学習計画修正の提案
生成完了時にメール通知する。

学習者はDay1診断完了後、いつでも週次・月次レビューの一覧をマイページ/コース画面から確認できる。

### 5.6 AI伴走の接点設計

Push型とPull型のハイブリッド構成を採用する。

| 種別 | 用途 | 実装 |
|---|---|---|
| Pull型(学習者が能動的にアクセス) | 学習相談・質問・学習報告 | Webチャット |
| Push型(AIが能動的に働きかける) | 朝・夜の声かけ、未開封リマインド | アプリ内通知 + メール通知 |

**朝・夜の声かけ通知**: 学習者が設定した時刻(デフォルト07:00/21:00)になると、当日のタスク・直近3日分のサマリーを踏まえたメッセージをその場でAI生成し、アプリ内通知とメールの両方で送信する(1日1コースあたり朝・夜各1通まで)。時刻判定は5分の許容誤差で行う。

**未開封リマインド(3段階)**: 学習者が最後にチャットを開いた(質問を送った)日から1日以上経過すると、経過日数に応じて段階的にトーンが変わるリマインドメールを送信する(1日1コースあたり1通まで)。4日以上未開封の場合は、担当クリエイターにも「学習者が連絡が取れていない」旨をメールで通知する。

通知時刻は学習者が自分で設定する(朝・夜それぞれ)。

**MVPで対応しないもの**: スマホアプリのネイティブプッシュ通知・LINE連携・Discord連携

### 5.7 相談・質問対応フロー

学習者からの相談への対応はティアによって異なる。

#### Tier A(AIのみ)

```
学習者が質問送信(1日30件・合計2000文字/日の上限あり)
  ↓
AIが質問を自動タグ付け(カテゴリ分類)
  ↓
未知のカテゴリ名はstatus='pending'の候補として作成され、クリエイターが承認するまで
コンテンツ紐付け・フラストレーション検知の対象にはならない
  ↓
紐付けコンテンツの検索(承認済みカテゴリのみ)
  ├── 該当あり → コンテンツ + クリエイター口調のメッセージを返信
  └── 該当なし → 人格プロファイルのみで回答生成
  ↓
質問ログを蓄積(分析ダッシュボードに反映)
```

同一カテゴリ(承認済みのもの)への質問が直近7日間で3回以上続くと「フラストレーション信号」を検知し、Tier Bへのアップグレードを学習者に提案する(事業検証ポイント①)。

#### Tier B(講師が1日1回直接回答)

```
学習者が質問送信(1日1回まで講師へ届く)
  ↓
AIが下書き回答を生成(常に高品質モデルを使用)
  ↓
講師にメール通知
  ↓
講師が管理画面で下書きを確認・編集(または無編集で承認)して送信
  ↓
24時間以上未対応の場合:
  管理者の監視ダッシュボード(G-04)・クリエイター自身の質問一覧で「未対応」として
  最優先表示される(自動メッセージの学習者への表示は未実装)
```

2回目以降の質問はAIが自動回答(Tier Aと同じフロー)。

### 5.8 質問分析ダッシュボード(クリエイター向け)

両ティア共通で、全質問を蓄積・分析してクリエイターに提供する。

**表示内容**
- 質問カテゴリ別ランキング(承認済みカテゴリのみ。質問数の多い順)
- カテゴリ別の実際の質問文一覧
- コンテンツ紐付け(カテゴリに動画/記事/PDFのタイトル・URLを登録、複数件登録・削除可能)
- 未承認カテゴリ一覧(AIが新規に提案したカテゴリ候補)の承認/却下UI。承認するまで分析・コンテンツ紐付けの対象に含まれない

**コンテンツ×質問の紐付けループ**

```
① 学習者から「仮定法」の質問が多く集まる
  ↓
② ダッシュボードで可視化される
  ↓
③ クリエイターが仮定法の動画を作成・投稿
  ↓
④ 管理画面で「仮定法」カテゴリに動画URLを紐付ける
  ↓
⑤ 以降、同種の質問が来たらAIが自動でその動画を含めて回答
  ↓
⑥ 使えば使うほどAIの回答品質が向上していく
```

### 5.9 決済フロー

#### 学習者側

```
コース選択(Tier A / Tier B) → Stripe決済(サブスク) → サブスク開始 → 毎月自動更新
```

- 解約: マイページから即時解約可能(Stripe側のサブスクリプションも同時に削除)
- Tierダウングレード/アップグレード: 解約・再契約せず、既存サブスクリプションを更新して即時切替可能(Stripe側はプロレーション計算)
- 決済失敗時(past_due)も3日間は猶予期間としてコース利用を継続できる

買い切り型コース(Tier設定のないコース)については、Stripe Payment Intentによる単発決済(`/payments/checkout`)も実装されている。

#### クリエイター側

```
ユーザー課金 → 収益ダッシュボードで集計表示(売上総額・プラットフォーム手数料(20%)・クリエイター取り分・アクティブ契約数)
```

> 補足: 収益の可視化(ダッシュボード)は実装済みだが、クリエイターへの実際の資金移動(Stripe Connect等を用いた月次振込)は本書執筆時点では未実装。

**MVPの対応範囲**
- AI伴走コース月額サブスクのみ(Tier A / Tier B)が中心。買い切り型コース購入機能も実装されている
- 都度課金・分割払いは対象外

**価格帯**
- Tier A(AIのみ): 980〜1,980円/月
- Tier B(プレミアム): 2,980〜5,000円/月

### 5.10 管理者機能(MVP最低限)

| # | 機能 |
|---|---|
| G-01 | クリエイター申請の審査・承認/却下、承認済みクリエイターの停止・再開 |
| G-02 | 違反コンテンツ・コースの停止/解除、コースの完全削除(在籍中の学習者がいる場合は不可) |
| G-03 | ユーザーからの通報管理(対応済みへの更新) |
| G-04 | Tier B講師の回答状況監視(全クリエイター横断、24時間未回答のアラート一覧) |
| G-05 | コース公開審査(`review`状態のコースを承認して`published`に、または却下して`draft`に戻す) |
| G-06 | 教材プリセット管理(教材名・出版社・種別・対象・目次の登録・編集・削除。コースで使用中の教材は削除不可) |
| G-07 | Stripe決済の返金処理(購入済みコースを全額返金) |
