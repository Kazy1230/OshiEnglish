# ManaVillage コミュニケーション設計 詳細仕様

> ステータス: ドラフト v1.1
> 関連ドキュメント: ManaVillage 要件定義書 v1.1 / Day1初回診断フロー詳細仕様

---

## 1. 概要

学習者からの相談・質問への対応は、コースのティアによって異なる。
どちらのティアでも質問は蓄積・分析され、クリエイターのコンテンツ改善に活用される。

---

## 2. コースティア定義

| ティア | 名称 | 回答者 | 価格帯 |
|---|---|---|---|
| Tier A | AI伴走コース | AIのみ | 980〜1,980円/月 |
| Tier B | プレミアム伴走コース | 講師が1日1回直接回答 | 2,980〜5,000円/月 |

---

## 3. 学習者からの相談の種類

| 種別 | 例 |
|---|---|
| ① 感情・モチベーション系 | 「単語のモチベがなくなっちゃって...」「続けられるか不安」 |
| ② 学習内容の質問系 | 「仮定法ってどう覚えればいい?」「Part 3が全然取れない」 |
| ③ 状況報告・雑談系 | 「今日30分やりました」「昨日できなかった...」 |

---

## 4. Tier A: AI伴走コースの回答フロー

```
学習者が質問を送信
  ↓
AIが質問を自動タグ付け(カテゴリ分類)
  ↓
紐付けコンテンツの検索
  ├── 該当コンテンツあり → コンテンツ+クリエイター口調のメッセージを返信
  └── 該当コンテンツなし → 人格プロファイルのみで回答生成
  ↓
質問ログを蓄積(分析ダッシュボードに反映)
```

### 回答生成の優先順位

1. **クリエイターが紐付けたコンテンツがある場合**
   - そのコンテンツ(動画URL・記事等)を提示
   - 人格プロファイルを適用した紹介メッセージを添える

   ```
   返信例:
   「仮定法、悩むよね。実はこれについて動画を作ったんだ。
   まずこれを見てみて！→ [仮定法マスター動画]
   見てわからなかったらまた聞いてね。」
   ```

2. **紐付けコンテンツがない場合**
   - 人格プロファイル + 学習者の診断結果をもとにAIが回答
   - 種別ごとに回答スタイルを変える

   | 種別 | 回答スタイル |
   |---|---|
   | ① 感情・モチベーション系 | まず共感→原因の整理→小さな行動提案 |
   | ② 学習内容の質問系 | 結論→理由→具体例→次のアクション |
   | ③ 状況報告・雑談系 | 労い→翌日への橋渡し |

---

## 5. Tier B: プレミアム伴走コースの回答フロー

```
学習者が質問を送信
  ↓
AIが質問を自動タグ付け(カテゴリ分類)
  ↓
AIが下書き回答を生成(人格プロファイル適用)
  ↓
講師に通知(メール + 管理画面バッジ)
  ↓
講師が管理画面で下書きを確認
  ├── 承認 → 即時送信
  ├── 編集して承認 → 編集内容を送信
  └── 24時間以内に未対応
        → 学習者に自動表示:
          「現在講師は忙しいです。次回までに回答します」
          → 次の講師ログイン時に優先通知
```

### 講師向け管理画面(回答UI)

| 要素 | 内容 |
|---|---|
| 未回答一覧 | 学習者名・質問内容・受信時刻・経過時間 |
| AI下書き | 人格プロファイルで生成した回答案 |
| 編集フィールド | 下書きを直接編集して送信 |
| 一括承認ボタン | AI下書きをそのまま送信(複数まとめて) |
| タグ表示 | 自動付与されたカテゴリタグ |

### 1日1回の回数制限について

- 学習者1人につき1日1回まで講師への質問が届く
- 2回目以降の質問はAIが自動回答(Tier Aと同じフロー)
- 「今日の講師への質問は使いました。AIに相談する場合はこちら」と表示

---

## 6. 質問分析ダッシュボード(クリエイター向け)

両ティア共通で、全質問を蓄積・分析してクリエイターに提供する。

### 表示内容

#### ① 質問カテゴリ別ランキング

```
【今週の質問Top5】
1位: 仮定法 (12件)
2位: モチベーション維持 (8件)
3位: Part 3 リスニング (7件)
4位: 単語の覚え方 (5件)
5位: 学習時間の確保 (4件)
```

#### ② カテゴリ別詳細

各カテゴリをクリックすると実際の質問文一覧を確認できる。

```
「仮定法」カテゴリの質問(12件)
- 「仮定法過去完了がいつも混乱します」
- 「If節の使い分けがわからない」
- 「仮定法ってどう覚えればいいですか?」
...
```

#### ③ コンテンツ紐付けボタン

質問カテゴリの横に「コンテンツを紐付ける」ボタンを設置。
クリックすると動画URL・記事URLを登録できる。

---

## 7. コンテンツ×質問の紐付けループ

ManaVillageの差別化を生む核心的な仕組み。

```
① 学習者が「仮定法」について多く質問する
  ↓
② ダッシュボードに「仮定法(12件)」と表示される
  ↓
③ クリエイターが仮定法の動画を作成してYouTube等に投稿
  ↓
④ 管理画面で「仮定法」カテゴリに動画URLを紐付ける
  ↓
⑤ 以降、「仮定法」に分類される質問が来たら
    AIが自動でその動画を含めて回答する
  ↓
⑥ 新しい質問が来るたびにループが繰り返される
   → クリエイターの知識が蓄積されてAIの回答品質が上がっていく
```

### 紐付けデータ構造

```
question_categories
  - id
  - creator_id
  - name          (例: "仮定法")
  - keywords      (JSON: 自動タグ付けに使うキーワード群)
  - created_at

category_contents
  - id
  - category_id
  - content_type  (video / article / pdf)
  - title
  - url
  - created_at
```

---

## 8. 質問の自動タグ付け仕様

### 分類方法

AIが質問文を解析し、以下の2軸で分類する。

| 軸 | 分類例 |
|---|---|
| 学習コンテンツ軸 | 仮定法・単語・リスニング・Part3・文法 等 |
| 感情・状態軸 | モチベーション・継続・不安・時間不足 等 |

### タグ付けロジック

1. クリエイターが設定した`question_categories`のキーワードと照合
2. 一致するカテゴリがあれば既存カテゴリに分類
3. 一致しない場合はAIが新規カテゴリ候補を提案
   → クリエイターが管理画面で「カテゴリとして登録する/しない」を判断

---

## 9. データモデル

```
questions
  - id
  - user_id          (学習者)
  - course_id
  - tier             (A / B)
  - body             (質問文)
  - category_id      (自動タグ付け結果)
  - status           (pending / answered_by_ai / answered_by_instructor / pending_instructor)
  - created_at

answers
  - id
  - question_id
  - answered_by      (ai / instructor)
  - body             (回答文)
  - linked_content_url  (紐付けコンテンツURL。あれば)
  - is_draft         (Tier Bの講師確認前はtrue)
  - created_at
  - sent_at
```
