# ManaVillage Day1 初回診断フロー詳細仕様

> ステータス: ドラフト v1.1
> 関連ドキュメント: ManaVillage 要件定義書 v1.1 / コース生成ワークフロー詳細仕様

---

## 1. 概要

Day1は学習者がManaVillageで最初に体験するフロー。
クリエイターのAIが学習者に寄り添い、現状把握→90日計画提示→通知設定まで一気通貫で完結する。

**所要時間目安**: 5〜10分

---

## 2. フロー全体像

```
Phase 1: ウェルカムメッセージ
  ↓
Phase 2: 診断チャット(6問)
  ↓
Phase 3: AIによる分析・90日ロードマップ生成
  ↓
Phase 4: 90日計画の提示
  ↓
Phase 5: 通知時刻の設定
  ↓
Phase 6: 本日のタスク提示
  ↓
Day1 完了
```

---

## 3. Phase 1: ウェルカムメッセージ

AIがクリエイターの人格で学習者に挨拶する。

**表示内容**
- クリエイターのアバター画像
- ウェルカムメッセージ(人格プロファイルを適用してAIが生成)

**メッセージ例(継続重視・優しめのクリエイターの場合)**
```
はじめまして！今日からあなたの英語学習を一緒に進めていきましょう。

まず、今のあなたの状況を教えてもらえますか？
いくつか質問するので、気軽に答えてくださいね。
```

---

## 4. Phase 2: 診断チャット(7問)

チャット形式でAIが1問ずつ質問する。
学習者はテキスト入力・数値入力・選択肢から回答する。

### 質問リスト

| # | 質問 | 回答形式 | 取得目的 |
|---|---|---|---|
| Q1 | 「現在のTOEICスコアを教えてください。」 | 数値入力 + 「未受験」チェックボックス | 現在地の把握。表記揺れを防ぐため数値と未受験を分離 |
| Q2 | 「目標スコアはいくつですか？」 | 数値入力 | ゴール設定 |
| Q3 | 「試験を受ける予定の時期を教えてください。」 | 選択肢 | 期間設定。テキスト入力より分析が安定する |
| Q4 | 「1日に英語学習に使える時間はどのくらいですか？」 | 選択肢 | 学習強度設定 |
| Q5 | 「英語で一番苦手だと感じる分野はどれですか？」 | 選択肢(複数可) | 弱点把握 |
| Q6 | 「これまで英語学習でどんなことをしてきましたか？」 | テキスト入力 | 学習歴の把握 |
| Q7 | 「今使っている(または使う予定の)教材があれば教えてください。」 | テキスト入力(任意) | AIが具体的な教材名を使ったタスク提示をするため |

### Q1の入力UI
- 数値入力フィールド(整数のみ)
- 「まだ受けたことがない」チェックボックス → チェックすると数値フィールドを非活性化
- これにより「未受験」「まだ」「なし」などの表記揺れを排除し、後の分析ロジックを安定させる

### Q3の選択肢
- 1ヶ月以内
- 2〜3ヶ月後
- 4〜6ヶ月後
- 半年以上先
- まだ決めていない

### Q4の選択肢
- 15分程度
- 30分程度
- 1時間程度
- 2時間以上

### Q5の選択肢(複数選択可)
- リスニング
- 文法
- 語彙・単語
- 読解
- 全体的に苦手

### チャットUI仕様
- AIのメッセージはクリエイターのアバター付きで左側に表示
- 学習者の回答は右側に表示
- 選択肢はボタン形式で表示(タップで選択)
- Q7は任意入力のため「スキップ」ボタンを表示
- 回答済みの質問は上にスクロールして確認可能(編集不可)

## 5. Phase 3: AIによる分析・90日ロードマップ生成

### ローディング表示

診断回答送信後、AIが分析中であることを示すアニメーションを表示。

```
表示メッセージ例:
「あなたの状況を分析しています...」
「90日間の学習計画を作成しています...」
「もう少しお待ちください...」
```

所要時間目安: 5〜15秒

### AIへの入力

以下を組み合わせてAIが90日ロードマップを生成する。

- 診断結果(Q1〜Q6の回答)
- クリエイターの人格プロファイル
- コース基本設定(ゴール・学習強度)
- course_daysテーブルに保存された90日コース構造

### 生成内容

クリエイターが設定した90日コース構造をベースに、
学習者の診断結果に応じて**強調ポイントと週次アドバイスをパーソナライズ**する。

※ 90日のカリキュラム自体は変更しない。
  学習者ごとに変わるのは「声かけの内容」「週次アドバイスの優先順位」のみ。

## 6. Phase 4: 90日計画の提示

### 表示構成

#### ① レベル分析サマリー

```
【あなたの現在地分析】

現在スコア: 580点
目標スコア: 800点
差分: +220点
試験まで: 約90日

得意: 読解
苦手: リスニング・語彙

▶ この差分は90日で達成可能な範囲です。一緒に頑張りましょう！
```

#### ① ロードマップの根拠(サマリー直下に表示)

「なぜこのロードマップになったのか」を1行で明示し、納得感を高める。

```
【このプランの理由】
読解はすでに得意なため、90日間でスコアを最大化できる
リスニングと語彙の強化に重点を置いたプランにしています。
```

AIが診断結果(得意・苦手・試験日・学習時間)をもとに自動生成する。

#### ② 90日ロードマップ(週単位)

| 期間 | テーマ | マイルストーン |
|---|---|---|
| Week 1〜2 | 基礎固め・学習習慣の確立 | 毎日30分の学習を継続 |
| Week 3〜6 | 語彙力強化・文法定着 | 単語1,000語習得 |
| Week 7〜10 | リスニング集中強化 | Part 1〜4の正答率70%以上 |
| Week 11〜12 | 実戦演習 | 模擬試験で750点達成 |
| Week 13 | 仕上げ・本番準備 | 本番想定の時間管理習得 |

#### ③ クリエイターからのメッセージ

人格プロファイルを適用してAIが生成。

```
メッセージ例:
「580点から800点、差分220点。一見大きく見えるかもしれないけど、
毎日コツコツ続ければ必ず届く数字です。

あなたの場合、リスニングと語彙が伸びれば
一番スコアが上がりやすいと分析できました。
まずはそこから一緒に攻略していきましょう！」
```

#### ④ アクションボタン

```
[この計画で始める] ← メインCTA
[計画の詳細を見る] ← 90日カレンダーを展開表示
```

**「計画の詳細を見る」押下時の表示**

カレンダーを単純に表示するのではなく、Day1〜Day3の具体的なタスクをチラ見えさせることで、学習開始のイメージを持たせてメインCTAへの転換率を高める。

```
【最初の3日間】

Day1: 診断完了 + 公式問題集 Part 1を1セット
Day2: 単語20語 + Part 2リスニング練習
Day3: 単語20語 + 文法基礎確認(関係代名詞)

▼ Day4以降はこの計画で始めてから確認できます

[この計画で始める]
```

## 7. Phase 5: 通知時刻の設定

### 設定項目

| 通知種別 | 内容 | デフォルト |
|---|---|---|
| 朝の声かけ | AIからの励ましメッセージ+今日のタスク通知 | 7:00 |
| 夜のリマインド | 学習報告の促し | 21:00 |

### UI

- 時刻はドラム式ピッカーまたはセレクトボックスで選択
- デフォルト時刻を表示する際に、その理由を一言添えて納得感を高める

```
朝 7:00(通勤・通学の時間帯に合わせたデフォルトです)
夜 21:00(就寝前の振り返りに合わせたデフォルトです)
```

- 「通知は後で設定する」でスキップ可能
- スキップした場合もデフォルト時刻(7:00/21:00)が適用され、設定完了画面でその旨を伝える

### 設定完了メッセージ

```
設定完了！
毎日 7:00 に声かけします。
学習報告は 21:00 にリマインドするね。

いつでもマイページから変更できるよ。
```

## 8. Phase 6: 本日のタスク提示

Day1のタスクをチャット形式で提示する。

### 表示内容

```
それじゃあ、今日から始めよう！

【今日のタスク】
① この診断チャットに答えてくれたこと → ✅ 完了！
② 公式問題集のPart 1を1セット解いてみよう
③ 単語アプリで20語学習

まず②から始めてみて。
終わったら「完了」と送ってね！
```

### タスク完了の報告

- 学習者が「完了」「やった」「できた」等を送信するとAIが反応
- AIがクリエイターの口調で労いのメッセージを返す

```
お疲れ様！初日をやり切ったね。
この調子で続けましょう。明日も一緒に頑張ろう！
```

## 10. データモデル

```
learner_profiles
  - id
  - user_id
  - course_id
  - current_score      (診断Q1)
  - target_score       (診断Q2)
  - exam_date          (診断Q3: 試験予定時期)
  - daily_study_time   (診断Q4: 学習時間)
  - weak_areas         (診断Q5: JSON配列)
  - study_history      (診断Q6: テキスト)
  - created_at

learner_roadmaps
  - id
  - learner_profile_id
  - level_analysis     (JSON: レベル分析サマリー)
  - week_advices       (JSON: 週次アドバイスのパーソナライズ内容)
  - created_at

notification_settings
  - id
  - user_id
  - course_id
  - morning_time       (例: "07:00")
  - evening_time       (例: "21:00")
  - is_enabled
  - updated_at

day_logs
  - id
  - user_id
  - course_id
  - day_number
  - is_completed
  - completed_at
  - memo               (任意: 学習メモ)
  - created_at
```

## 12. 将来的な拡張(Phase 2以降)

**学習開始30日目の自動再診断**

`learner_profiles`に構造化された診断データが保存されているため、将来的に以下の実装が可能。

- 学習開始30日目・60日目に自動で再診断を促す通知を送信
- AIとの会話の中で自然にスコアの近況を聞き出す(例:「そういえば最近スコアどう?」)
- 再診断結果をもとに`learner_roadmaps`を更新し、残り期間の計画を修正する

これにより伴走の質が継続的に向上し、解約率の低下にも貢献する。
