# ManaVillage コース生成ワークフロー詳細仕様

> ステータス: ドラフト v1.0
> 関連ドキュメント: ManaVillage 要件定義書 v1.0

## 1. ワークフロー全体像

```
Step 1: AIインタビュー(人格収集)
  ↓
Step 2: 人格プロファイル生成・確認
  ↓
Step 3: コース基本情報入力
  ↓
Step 4: 90日コース自動生成
  ↓
Step 5: クリエイターによる日単位編集
  ↓
Step 6: 参考資料の添付
  ↓
Step 7: プレビュー・公開申請
  ↓
Step 8: 運営確認 → 公開
```

## 2. Step 1: AIインタビュー(人格収集)

### 設計方針

- 5問の固定質問でサクッと終わる設計
- 回答内容に応じてAIが深掘り質問を最大3問追加(適応型)
- 合計: 最短5問、最長8問程度
- 所要時間の目安: 10〜15分

### 固定質問リスト(英語学習領域)

| # | 質問 | 目的 |
|---|---|---|
| Q1 | 「学習者が挫折しそうになったとき、どのように声をかけますか?」 | 励まし方・コミュニケーションスタイルの把握 |
| Q2 | 「TOEIC初心者に最初の1週間で何をさせますか?その理由も教えてください。」 | 指導の優先順位・学習哲学の把握 |
| Q3 | 「単語・文法・リスニング・読解のうち、最も重視するものはどれですか?なぜですか?」 | 指導方針・判断基準の把握 |
| Q4 | 「あなたの指導を受けた学習者が成果を出せないとき、原因はどこにあると思いますか?」 | 問題解決アプローチの把握 |
| Q5 | 「あなたの指導で一番大切にしていることを一言で表すとしたら何ですか?」 | 学習哲学・核心的価値観の把握 |

### 深掘り質問(AIが自動生成)

回答が短い・抽象的な場合にAIが追加質問を生成する。

例:
- Q1への回答が「頑張れと伝えます」→「具体的にどんな言葉を使いますか?実際のセリフを教えてください。」
- Q3への回答が「文法です」→「文法のどの部分から始めますか?なぜそこからですか?」

### インタビューUI

- チャット形式で1問ずつ表示
- クリエイターがテキストで自由回答
- 「次の質問へ」ボタンで進む
- 途中保存可能(ブラウザを閉じても続きから再開できる)

### 確認画面

- AIが生成した人格プロファイルをクリエイターが確認
- 修正したい項目があれば自由テキストで上書き可能
- 「この内容でコース生成に進む」ボタンで次へ

|---|---|---|

## 5. Step 4: 90日コース自動生成

### 生成内容の構造

90日を13週に分割し、週単位のテーマと日単位の内容を生成する。

```
コース
  ├── Week 1〜2: 基礎固め期
  │     ├── 週テーマ・マイルストーン
  │     └── Day 1〜14: 各日の内容
  ├── Week 3〜8: 実力養成期
  ├── Week 9〜12: 実戦演習期
  └── Week 13: 仕上げ・本番準備期
```

### 1日分の生成内容

```json
{
  "day": 1,
  "theme": "学習スタート・現在地の確認",
  "tasks": [
    "初回診断チャットに答える",
    "単語帳アプリで20語学習",
    "公式問題集パート1を1セット解く"
  ],
  "ai_message": {
    "morning": "今日からスタートだね！まず今の実力を一緒に確認しよう。焦らなくて大丈夫、小さな一歩から始めましょう。",
    "evening_reminder": "今日の学習はできた？報告待ってるよ。",
    "completion": "お疲れ様！初日をやり切ったね。この調子で続けましょう。"
  },
  "is_rest_day": false,
  "week_number": 1
}
```

### 生成に使うインプット

- 人格プロファイル(Step 2)
- コース基本情報(Step 3: ゴール・対象者・学習強度)
- 英語学習の一般的な90日カリキュラム知識(AIの学習データ)

※ 教材(PDF・URL)はコース生成には使用しない(Step 6で別途添付)

### 生成時間

- AIによる90日分の一括生成
- 生成中はプログレスバーで進行状況を表示
- 目安: 30〜60秒

## 6. Step 5: クリエイターによる日単位編集

### 編集画面の設計

**カレンダービュー**
- 90日分をカレンダー形式で一覧表示
- 各日をクリックすると詳細編集パネルが開く
- AI生成済みは青、クリエイター編集済みは緑で色分け

**1日分の編集パネル**

| 項目 | 編集可否 |
|---|---|
| 日のテーマ | ○ 自由編集 |
| タスクリスト | ○ 追加・削除・並び替え |
| AIメッセージ(朝) | ○ 自由編集 |
| AIメッセージ(夜リマインド) | ○ 自由編集 |
| AIメッセージ(完了時) | ○ 自由編集 |
| 休息日フラグ | ○ ON/OFFで切り替え |

**一括操作**
- 「週単位でテーマを変更」
- 「全日のAIメッセージを口調統一」(人格プロファイルを再適用)
- 「特定の週を休息週に設定」

## 7. Step 6: 参考資料の添付

### 方針

教材はコース内容とは独立して「参考資料」として添付するのみ。
コース生成への組み込みは行わない。

### 添付できるもの

| 種別 | 内容 |
|---|---|
| PDF | 単語リスト・問題集・オリジナルテキスト等 |
| URL | 参考サイト・YouTube動画・ブログ記事等 |

### 表示場所

学習者のコースページの「参考資料」セクションに一覧表示。
日単位のコンテンツとは別の場所に配置。

---

## 8. Step 7: プレビュー・公開申請

### プレビュー画面

- 学習者目線でコースを確認できる
- Day1の初回診断フロー・チャット画面のサンプルを確認可能

### 公開申請

- 「公開申請」ボタンを押すと運営に通知
- 申請時のチェック項目:
  - コース名・ゴールが入力されている
  - 90日分の内容が生成されている
  - 価格が設定されている

---

## 9. Step 8: 運営確認 → 公開

### 運営確認項目

| 確認項目 | 内容 |
|---|---|
| コンテンツ適切性 | 誤情報・有害な内容がないか |
| 価格設定 | 980〜2,980円の範囲内か |
| クリエイター本人確認 | 申請時のSNSと一致しているか |

### 承認フロー

- 運営が管理画面から「承認」または「差し戻し」
- 差し戻しの場合は理由をクリエイターにメール通知
- 承認後はマーケットプレイスに自動掲載

## 10. データモデル概要

```
creator_personality_profiles
  - id
  - creator_id
  - interview_answers (JSON: 質問と回答のペア)
  - profile (JSON: 人格プロファイル構造体)
  - created_at

courses
  - id
  - creator_id
  - personality_profile_id
  - title
  - goal
  - target_learner
  - intensity (1日の学習時間目安)
  - price
  - status (draft / review / published / suspended)
  - created_at

course_days
  - id
  - course_id
  - day_number (1〜90)
  - week_number (1〜13)
  - theme
  - tasks (JSON: タスクリスト)
  - ai_message_morning
  - ai_message_evening
  - ai_message_completion
  - is_rest_day
  - is_edited_by_creator
  - created_at
  - updated_at

course_materials
  - id
  - course_id
  - type (pdf / url)
  - title
  - file_url
  - created_at
```
