# ManaVillage 要件定義書 v1.1

> ステータス: ドラフト v1.1
> 対象フェーズ: MVP(英語学習 AI伴走コース)
> 更新内容: 2ティア制・質問分析ダッシュボード・コンテンツ紐付けループを追加

---

## 1. サービス概要

### サービス名
ManaVillage

### コンセプト
**成果達成型メンタープラットフォーム**

クリエイターの学習哲学・指導方針・判断基準をAIが再現し、学習者に伴走体験を提供するプラットフォーム。

### ミッション
「誰もがメンターを持てる世界をつくる」

### MVPの検証仮説
「ユーザーは動画教材ではなく、伴走体験にお金を払うのか？」

---

## 2. MVPのスコープ

### 対象ジャンル
英語学習(TOEIC / 英検 / 留学)

### 対象クリエイター
英語系マイクロクリエイター(フォロワー300〜5,000人)

### 対象学習者
TOEIC学習者 / 英検受験者 / 留学希望者

### MVPで実装しないもの
- Tier1 コンテンツ単体販売
- Tier3 本人メンタリング
- コミュニティ機能
- スマホアプリ / プッシュ通知
- LINE・Discord連携
- 複数ジャンル展開

---

## 3. ロール定義

| ロール | 概要 |
|---|---|
| 学習者 | コースを購入し、AI伴走を受ける |
| クリエイター | コースを作成・販売する |
| 管理者 | プラットフォームを管理する |

---

## 4. コースティア定義

MVPでは2種類のコースティアを提供する。

| ティア | 名称 | 回答者 | 価格帯 |
|---|---|---|---|
| Tier A | AI伴走コース | AIのみが回答 | 980〜1,980円/月 |
| Tier B | プレミアム伴走コース | AIに加え講師が1日1回直接回答 | 2,980〜5,000円/月 |

クリエイターはどちらのティアを提供するか選択できる。両方提供することも可能。

### 5.2 クリエイター申請フロー

初期フェーズでは品質管理を優先し、運営審査制を採用する。

| Step | 内容 |
|---|---|
| Step1 | クリエイター登録申請 |
| Step2 | プロフィール入力(名前・専門分野・指導実績) |
| Step3 | SNSアカウント提出(X / YouTube / Instagram / ブログ等) |
| Step4 | 運営審査(実在性・専門性・ガイドライン適合性を確認) |
| Step5 | 承認 → コース作成権限付与 |

---

### 5.3 人格データ収集(AIインタビュー)

フォーム入力ではなく、AIが対話形式でクリエイターにインタビューし人格データを抽出する。

**フロー**
```
Step1: AIがクリエイターに指導シナリオ質問(固定5問)
      例:「TOEIC学習者が単語学習を嫌がっています。どう指導しますか?」
Step2: クリエイターが自由回答
Step3: AIが必要に応じて深掘り質問(最大3問追加)
Step4: AIが人格プロファイルを生成
Step5: クリエイターが確認・修正して確定
```

所要時間目安: 10〜15分

**抽出項目**

| カテゴリ | 項目 |
|---|---|
| コミュニケーション | 口調・一人称・語尾 |
| 指導スタイル | 厳しさ・励まし方・フィードバック方法 |
| 学習哲学 | 重視する考え方・判断基準・優先順位 |
| 思考特性 | 例え話の癖・説明方法・問題解決アプローチ |

---

### 5.4 コース作成機能

**クリエイターの入力**

| カテゴリ | 項目 |
|---|---|
| 基本情報 | 名前・プロフィール・専門分野 |
| コース情報 | コース名・ゴール・対象者・コースティア(A/B) |
| 指導方針 | 学習哲学・重視する考え方・よく使う表現 |
| 参考資料 | PDF・URL(コース生成には使用せず参考資料として添付) |

**AIが自動生成する内容**
- コース概要
- 90日学習計画(週単位テーマ + 日単位タスク・声かけメッセージ)
- AI伴走設定(人格プロファイルを適用)

クリエイターはAI生成結果を日単位で確認・編集してから公開する。

---

### 5.5 AI伴走コース(コア機能)

ManaVillageの最重要機能。

#### Day1: 初回診断(チャット形式)

| 取得情報 | 内容 |
|---|---|
| 現在スコア | 現時点でのTOEICスコア等 |
| 目標スコア | 達成したいスコア |
| 試験日 | 受験予定時期 |
| 学習可能時間 | 1日あたりの学習時間 |
| 苦手分野 | リスニング・文法・語彙等(複数選択可) |
| 学習歴 | これまでの学習経験 |

AIが実施:
- レベル分析
- 90日ロードマップ生成(診断完了後すぐに提示)
- 通知時刻の設定(学習者が朝・夜の時刻を設定)

#### Day2〜90: 日次伴走

| 機能 | 内容 |
|---|---|
| 学習報告 | 当日の学習内容を報告 |
| 相談・質問対応 | 学習上の疑問・悩みをAIに相談 |
| モチベーション支援 | AIが励まし・継続支援 |

#### 週次レビュー

AIが分析: 学習量・継続率・苦手分野
出力: フィードバック + 翌週の学習計画

#### 月次レビュー

- 目標との差分確認
- 学習計画修正

### 5.6 AI伴走の接点設計

Push型とPull型のハイブリッド構成を採用する。

| 種別 | 用途 | 実装 |
|---|---|---|
| Pull型(学習者が能動的にアクセス) | 学習相談・質問・学習報告 | Webチャット |
| Push型(AIが能動的に働きかける) | リマインド・モチベーション維持 | メール通知 |

通知時刻は学習者が自分で設定する(朝・夜それぞれ)。

**MVPで対応しないもの**: スマホアプリ・プッシュ通知・LINE連携・Discord連携

---

### 5.7 相談・質問対応フロー

学習者からの相談への対応はティアによって異なる。

#### Tier A(AIのみ)

```
学習者が質問送信
  ↓
AIが質問を自動タグ付け(カテゴリ分類)
  ↓
紐付けコンテンツの検索
  ├── 該当あり → コンテンツ + クリエイター口調のメッセージを返信
  └── 該当なし → 人格プロファイルのみで回答生成
  ↓
質問ログを蓄積(分析ダッシュボードに反映)
```

#### Tier B(講師が1日1回直接回答)

```
学習者が質問送信(1日1回まで講師へ届く)
  ↓
AIが下書き回答を生成
  ↓
講師に通知(メール + 管理画面バッジ)
  ↓
講師が下書きを確認・編集して承認 → 送信
  ↓
24時間以内に未対応の場合:
  「現在講師は忙しいです。次回までに回答します」を自動表示
  → 次の講師ログイン時に優先通知
```

2回目以降の質問はAIが自動回答(Tier Aと同じフロー)。

---

### 5.8 質問分析ダッシュボード(クリエイター向け)

両ティア共通で、全質問を蓄積・分析してクリエイターに提供する。

**表示内容**
- 質問カテゴリ別ランキング(例:「仮定法 12件」「モチベーション 8件」)
- カテゴリ別の実際の質問文一覧
- コンテンツ紐付けボタン(カテゴリに動画・記事URLを登録)

**コンテンツ×質問の紐付けループ**

```
① 学習者から「仮定法」の質問が多く集まる
  ↓
② ダッシュボードで可視化される
  ↓
③ クリエイターが仮定法の動画を作成・投稿
  ↓
④ 管理画面で「仮定法」カテゴリに動画URLを紐付ける
  ↓
⑤ 以降、同種の質問が来たらAIが自動でその動画を含めて回答
  ↓
⑥ 使えば使うほどAIの回答品質が向上していく
```

### 5.9 決済フロー

#### 学習者側

```
コース選択(Tier A / Tier B) → 決済 → サブスク開始 → 毎月自動更新
```

- 解約: マイページから即時解約可能

#### クリエイター側

```
ユーザー課金 → 手数料控除 → クリエイター残高 → 月次振込
```

**MVPの対応範囲**
- AI伴走コース月額サブスクのみ(Tier A / Tier B)
- 都度課金・分割払いは対象外

**価格帯**
- Tier A(AIのみ): 980〜1,980円/月
- Tier B(プレミアム): 2,980〜5,000円/月

## 6. 非機能要件

### 6.1 セキュリティ

| # | 要件 |
|---|---|
| S-01 | HTTPS通信の強制(SSL/TLS) |
| S-02 | パスワードのハッシュ化(bcrypt) |
| S-03 | CSRF対策 |
| S-04 | Stripe決済利用(カード情報は自社サーバーに保存しない) |
| S-05 | 個人情報の保存範囲を最小化(メールアドレス・学習履歴・課金情報のみ) |

### 6.2 AI関連

- AI生成コンテンツであること、および成果を保証しないことを利用規約・画面上に明示する

### 6.3 パフォーマンス

- ページ初期表示3秒以内
- AIチャットのレスポンス: ストリーミング表示で体感速度を改善

## 8. 成功判定基準(MVP)

### 学習者側
- 継続率
- 利用頻度(週あたりのチャット回数)
- 課金継続率(月次)

### クリエイター側
- コース作成率(申請承認後にコースを作成する割合)
- コース公開率
- 売上発生率

### 事業側
- 月額課金の成立件数
- 解約率
- LTV
