## 1. 優先①: フロントエンドの追従修正(変更v1.md セクション8の解消)

`is_admin` / `is_preset` / `credit_balance` / `unlock_cost` を参照していた18ファイルすべてを修正した。

### 1.1 `is_admin` → `role === 'admin'`

対象: `app/admin/page.tsx`、`app/change-password/page.tsx`、`app/login/page.tsx`、`app/chat/page.tsx`、
`app/purchases/page.tsx`、`app/rewards/page.tsx`、`app/shelf/page.tsx`、
`app/admin/tabs/{ArticlesTab,CharactersTab,CustomersTab,DashboardTab,LogsTab,OrdersTab}.tsx`

- `Me`型の`is_admin: boolean`を`role: string`に変更
- `u.is_admin` → `u.role === "admin"`、`!c.is_admin`(顧客一覧の管理者除外フィルタ) → `c.role !== "admin"`
- `CustomersTab.tsx`の顧客新規作成フォームの`is_admin: boolean`フィールドを`role: string`(learner/instructor/admin)に変更

### 1.2 `is_preset` → `instructor_id` の有無で判定

- [lib/theme.ts](../frontend/lib/theme.ts): `CharacterTheme.is_preset?: boolean` を `instructor_id?: number | null` に置き換え、
  判定用ヘルパー `isOfficialCharacter(theme)`(`instructor_id != null`を返す)を追加
- `app/shelf/page.tsx`: ウェルカムカード表示条件・Instagram案内表示条件を`isOfficialCharacter()`に置き換え
- `app/admin/tabs/CharactersTab.tsx`: 「公式キャラクターにする」チェックボックスを廃止し、
  「講師プロフィールID(instructor_profiles.id)」を直接入力する数値フィールドに変更。バッジ表示・保存ペイロードも追従
- `app/admin/tabs/ArticlesTab.tsx`: ウェルカムページのカテゴリ分類・対象キャラ選択リストの`is_preset`判定を
  `instructor_id == null`(オリジナル) / `!= null`(公式)に置き換え
- `app/admin/tabs/RewardsTab.tsx`: 隠しセリフ登録上限の判定が、バックエンド側で暫定的に全キャラ上限15件
  ([rewards.py](../backend/app/routers/rewards.py)参照)に変更されたことに合わせ、フロントエンドも上限を15に統一
  (講師ごとの上限差別化はStep3で再設計)

### 1.3 `credit_balance` / `unlock_cost` の削除

- `app/chat/page.tsx` / `app/shelf/page.tsx`: クレジット残高バッジ・「クレジット残高」ボタンを削除
- `app/admin/tabs/CustomersTab.tsx`: クレジット残高表示・クレジット残高手動調整UI(`handleAdjustCredits`等)を削除
- `app/admin/tabs/ServiceMenuTab.tsx`: 「定期便の開封コスト」設定フォーム(廃止された`/credit-settings`APIを呼んでいた)を削除
- `app/articles/[id]/page.tsx` / `app/shelf/page.tsx`: 記事の開封課金UI(ロック画面・「🔓 nクレジットを使って読む」ボタン・
  クレジット不足時の購入導線)を削除。バックエンドの`locked`は常に`false`を返すため実質常に開封済み表示になる
- [lib/api.ts](../frontend/lib/api.ts): 廃止されたバックエンドエンドポイントに対応する関数を削除
  (`purchaseCredits`, `adminGetCreditSettings`, `adminUpdateCreditSettings`, `adminAdjustCredits`)

### 1.4 `/credits` ページの削除

`app/credits/page.tsx`とその配下を削除。残存していた`/credits`への遷移リンク(`shelf`・`chat`の旧クレジット残高ボタン)も
合わせて削除済みのため、デッドリンクは残っていない。

## 2. 優先②: 認証のロール拡張(Step2)

### 2.1 JWTペイロードへのrole追加

変更v1.mdのStep1対応時点で既に実装済み([auth.py:98](../backend/app/routers/auth.py)):

```python
token = create_access_token({"sub": str(user.id), "role": user.role})
```

詳細設計書Section 1.1のペイロード仕様(`sub` / `role` / `exp`)に準拠。

### 2.2 `get_current_admin` の確認

[security.py](../backend/app/core/security.py)の`get_current_admin`は変更v1.mdで既に`role == "admin"`判定に
変更済みであることを確認した。JWTのroleクレームを信用せず、`get_current_user`がDBから現在のCustomerレコードを
都度取得して`role`を検証する設計のため、ロール変更が即時反映される(JWT再発行を待たない)。

今回追加で、将来の講師向けエンドポイント(`/studio/*`等)に備えて`get_current_instructor_or_admin`
(role が instructor または admin のみ許可)を新設した。現時点ではどのルーターからも未使用。

### 2.3 フロントエンドのロールベースルートガード

- [lib/auth.ts](../frontend/lib/auth.ts): `Role`型(`"learner" | "instructor" | "admin"`)、
  `hasRole(role, allowed)`、`isInstructorOrAbove(role)`ヘルパーを追加
- [lib/useRoleGuard.ts](../frontend/lib/useRoleGuard.ts): クライアントコンポーネント用の共通ルートガードフック。
  - 未ログイン → `/login`
  - ログイン済みだが許可ロールでない → 指定のリダイレクト先(既定`/shelf`)
  - 許可ロールであれば`/auth/me`のレスポンスを返す
- `/admin/*`: 既存の`app/admin/page.tsx`にて`role !== "admin"`なら`/shelf`へリダイレクト(既存ロジックを
  is_admin→roleに置換する形で対応済み、1.1節参照)
- `/dashboard/*`, `/studio/*`: 新規に`app/dashboard/page.tsx`・`app/studio/page.tsx`を作成し、
  `useRoleGuard(["instructor", "admin"])`で保護。現時点では「準備中」のプレースホルダー表示のみ
  (キャラクタービルダー・コンテンツ生成スタジオの実装はStep3以降)

### 起動時に発見・修正した不具合

`main.py`内で`_migrate_legacy_characters_to_instructor()`の呼び出しが、レガシーキャラクター
(白河雪菜・蒼井零等)のレコードを作成する`_ensure_preset_characters()`より**前**の位置にあったため、
フレッシュなDBでは移行対象のキャラクターがまだ存在せず、`instructor_id`の割り当てが一切行われない不具合が
あった。`_migrate_legacy_characters_to_instructor()`の呼び出しを`_ensure_preset_characters()`の**後**に
移動し、再起動後に全キャラクターが`manavillage_official`講師プロフィールへ正しく割り当てられることを確認した。

## 4. 既知の制約・残課題

- `/dashboard`・`/studio`はガード付きのプレースホルダーページのみで、実際のキャラクタービルダー・
  コンテンツ生成スタジオ機能はまだ実装されていない(修正点.md Step3〜5で対応予定)
- `RewardsTab.tsx`の隠しセリフ登録上限(15件)は、バックエンド側が`is_preset`廃止に伴い暫定的に
  全キャラ一律で適用している値と合わせた一時的な措置。講師ごとの上限差別化の仕組みはStep3で再設計が必要
- `CharactersTab.tsx`の講師プロフィールID入力は、選択UI(ドロップダウン)ではなく数値の直接入力に
  簡略化した。`instructor_profiles`一覧APIが整備された段階で選択式UIに置き換えるのが望ましい
- 記事の開封課金UI・クレジット残高表示はすべて削除したが、`Article.opened_at`・`Message.credit_cost`等の
  DBカラム自体は変更v1.mdの方針通り当面残置されている(Step3でコース/レッスン購入フローに統合予定)
