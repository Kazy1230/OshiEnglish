# 推しEnglish — サービス概要

## 1. サービス基本情報

| 項目 | 内容 |
|---|---|
| サービス名 | 推しEnglish |
| ドメイン | oshi-english.life |
| キャッチコンセプト | 「推しキャラと一緒に英語を勉強する」 |
| ターゲット層 | TOEIC・英検・IELTS・TOEFLなど英語学習に取り組む社会人／学生、およびアニメ・キャラクターコンテンツが好きな「オタク層」 |
| サービス形態 | キャラクターIP × 英語学習コーチング（チャット形式のオンライン家庭教師サービス） |

---

## 2. コンセプト・体験設計

- 顧客はそれぞれ専属の「キャラクター」（ツンデレ先輩、毒舌だが優しい先輩、ロボット先生など）を選び、そのキャラクターとDM（チャット）でやり取りする。
- やり取りを通じて、文法解説記事・各種試験対策の演習問題・ライティング／スピーキング添削などの学習コンテンツを受け取る。
- 学習内容そのものは一般的な英語教材と大差ないが、以下の「推し育成」要素によって学習の継続モチベーションを高める：
  - キャラクターが世界観のある口調・態度でメッセージを送ってくる
  - やり取りを重ねるごとに「親密度」が上がり、関係性が変化していく（敬語 → タメ口 → あだ名で呼んでくれる、など）
  - 親密度や記事依頼数に応じて、隠しセリフ・称号・壁紙などの「ご褒美」が解放される
- コンテンツは運営者がLLM（Claude）で下書きを生成し、確認・編集してから配信する「半自動運用」。「キャラクターが個人のために書いてくれた」という特別感を演出する。

### オリジナルキャラクター（is_preset=False）

- 顧客ごとに専属キャラクターをオーダーメイドで作成可能。

- サンプル例：ドラえもん風の優しいロボット先生、毒舌だが面倒見のいい先輩「鬼島先輩」など。

### Instagram連携

各公式キャラクターに公式Instagramアカウントが紐づいており、チャット画面・本棚からフォローを促す導線を設置。世界観演出・ファン化施策の一環。

## 4. 料金・クレジット制度

### 基本ルール

- **1クレジット ＝ 1円**として、500クレジット単位でStripe経由で購入可能（クレジットカード／Apple Pay／Google Pay対応）。
- クレジットは以下の用途に消費される：

| 用途 | 消費クレジット |
|---|---|
| キャラクターへのDM送信 | 1クレジット |
| 記事・問題のリクエスト | 依頼時に50クレジット＋記事が届いて本棚で開封する時に残り150〜350クレジット（合計200〜400クレジット） |
| キャラクターから届く特別記事（定期便）の開封 | 50クレジット（届くこと自体は無料） |

### クレジットの入手方法

| 方法 | 付与クレジット |
|---|---|
| 申し込み時：公式キャラクター選択 | キャラ作成費無料＋20クレジット |
| 申し込み時：オリジナルキャラクター作成（¥500） | 500クレジット付与（実質チャージ） |
| デイリーログインボーナス | 1日10クレジット（残高が50クレジットを超えない範囲でのみ付与） |
| クレジット購入 | 500クレジット単位でStripe決済 |

### 記事・問題リクエストの料金表（消費クレジット）

| 試験・種別 | パート | クレジット |
|---|---|---|
| TOEIC | リーディング（短文穴埋め・長文穴埋め・長文読解）各 | 200 |
| TOEIC | リスニング | 200 |
| TOEIC | ライティング／スピーキング | 各400 |
| 英検 | リーディング・リスニング 各パート | 200 |
| 英検 | ライティング・スピーキング | 各400 |
| IELTS | Reading / Listening | 各200 |
| IELTS | Writing Task1 | 200 |
| IELTS | Writing Task2 | 400 |
| IELTS | Speaking Part1〜3 | 各400 |
| TOEFL | Reading / Listening | 各200 |
| TOEFL | Writing Integrated | 400 |
| TOEFL | Writing Academic Discussion | 200 |
| TOEFL | Speaking Task1〜4 | 各400 |
| 文法記事（2,500〜3,000文字） | 1項目 | 200 |
| 追加ユニット（記事） | 1本 | 200〜（種別により変動） |

### 2段階課金の仕組み

記事・問題のリクエストは「依頼するハードル」を下げるため2段階で課金する：

1. **依頼時**：固定50クレジットのみ消費
2. **記事が届いて本棚で開封する時**：残りのクレジット（200クレジットの記事なら150、400クレジットの記事なら350）を消費

未開封の記事は本棚に🔒マーク付きで表示され、開封ボタンを押すとクレジットが消費されて以後いつでも読めるようになる。クレジット不足の場合は購入ページへの導線が表示される。

### 定期便記事（特別記事）

- 管理者が気分で用意した記事が、3日～5日ごとに1本、無料で本棚に自動配布される。
- 記事が届くこと自体は無料。本棚で開封して読むには50クレジットを消費する。

## 7. ゲーミフィケーション設計

### 親密度（intimacy）システム

累計ポイントに応じて6段階のレベル・呼び方が変化する：

| レベル | 必要ポイント | 呼び方の段階 |
|---|---|---|
| 0 | 0 | 敬語期 |
| 1 | 15 | やわらぎ期 |
| 2 | 40 | タメ口期 |
| 3 | 80 | あだ名期 |
| 4 | 140 | 親友期 |
| 5 | 220 | 特別な存在期 |

ポイント加算条件（管理画面から調整可能）：
- メッセージ送信時：+1
- ログイン時（1日1回まで）：+1
- コンテンツ購入時：+10
- 演習問題提出時：+1

### 報酬（リワード）システム

- カテゴリ：隠しセリフ（line）／称号（title）／壁紙（wallpaper）
- 解放トリガー：親密度レベル到達（1〜5）／記事依頼回数の累計到達
- 公式キャラクター限定の報酬も用意され、公式キャラを選ぶインセンティブの一つになっている
- 解放時はアニメーション演出付き

### 記事リクエスト × 達成のループ

1. 顧客が本棚／DMから記事・問題・添削をリクエスト
2. 管理者が記事を作成・編集し公開
3. 公開時にキャラクターから自動DM通知が届く
4. 元のリクエストの状態が自動で「完了」になる
5. 記事依頼回数・親密度ポイントの増加が新たな報酬解放のトリガーになる

## 9. 運営フロー（半自動運用）

1. 申し込みフォーム（/apply）から注文（Order）が作成される
2. 公式キャラクター選択時：キャラ作成費無料のためStripe決済をスキップし、アカウントを即時発行
3. オリジナルキャラクター選択時：Stripe Checkout決済 → Webhook経由でアカウント自動発行・通知メール送信
4. 完了画面（/apply/complete）でID・パスワードを一度だけ表示
5. すべての顧客向けコンテンツ・DM返信は、LLM（Claude）が下書きを生成し、運営者が確認・編集・トーン調整した上で公開・送信
6. 担当者割り当て・優先度管理により、複数の運営者・オペレーターで顧客対応を分担
7. 退会・返金リクエストにも対応（セルフ退会・管理者による全額返金）
